## 📋 版本目标

在 v1.0 的基础上，增加用户认证和数据云端存储功能，提升用户体验和数据持久化能力。


### 2. Markdown 在线存储功能

#### 2.1 功能特性
- ✅ 保存当前提取的商品信息到云端
- ✅ 基于 ASIN 作为唯一标识
- ✅ 同一个 ASIN 多次保存时更新（创建历史版本）
- ✅ 查看已保存的 Markdown 列表
- ✅ 从云端下载 Markdown 文件
- ✅ 删除已保存的 Markdown
- ✅ 搜索已保存的文档（按标题、ASIN）
- ✅ 同步状态显示（上传中、成功、失败）

#### 2.2 文件命名规则
```
格式: [ASIN].md
示例: B08XYZ123.md

冲突处理: 同一ASIN多次保存时，创建新版本
- 主文档: B08XYZ123.md
- 历史版本: B08XYZ123_v1.md, B08XYZ123_v2.md
```

#### 2.3 存储策略
```
方案A: 存储在 Appwrite Database（推荐）
- 优点: 易于搜索、查询、版本管理
- 缺点: 单文档大小限制（通常 1MB）

方案B: 存储在 Appwrite Storage + Database 索引
- 优点: 支持大文件
- 缺点: 查询需要两次请求

方案C: 混合方案
- Database 存储元数据和索引
- Storage 存储实际文件内容
```

## 📝 详细功能设计

### 4.1 用户认证流程

#### 注册流程
```
1. 用户打开扩展
2. 检测到未登录
3. 显示"注册/登录"按钮
4. 点击"注册" → 打开注册表单
5. 输入邮箱、密码、姓名
6. 调用 Appwrite Account.create()
7. 成功 → 自动登录并保存 Session
8. 失败 → 显示错误信息
```

#### 登录流程
```
1. 点击"登录"按钮
2. 打开登录表单
3. 输入邮箱、密码
4. 调用 Appwrite Account.createEmailPasswordSession()
5. 成功 → 保存 Session，更新UI
6. 失败 → 显示错误信息
```

#### 退出登录
```
1. 点击"退出登录"
2. 调用 Appwrite Account.deleteSession('current')
3. 清除本地存储
4. 更新UI为未登录状态
```

### 4.2 Markdown 保存流程

```
1. 用户在 Amazon 商品页面提取数据
2. 点击"保存到云端"按钮
3. 检查登录状态
4. 未登录 → 提示先登录
5. 已登录 → 提取 ASIN（从URL）
   - URL 格式: https://www.amazon.com/dp/B08XYZ123
   - ASIN: B08XYZ123
6. 检查云端是否已存在该 ASIN
7. 存在 → 提示"更新还是创建新版本"
8. 生成 Markdown 内容
9. 调用 Appwrite Database.createDocument()
10. 显示上传进度
11. 成功 → 添加到已保存列表
12. 失败 → 显示错误信息
```

### 4.3 Markdown 列表和管理

#### 查看已保存列表
```
- 显示云端所有已保存的文档
- 包含：标题、ASIN、保存时间、文件大小
- 支持分页或无限滚动（如果文档很多）
- 支持搜索和过滤
```

#### 下载 Markdown
```
- 点击"下载"按钮
- 从 Database 读取 content 字段
- 触发浏览器下载
- 文件名: [ASIN].md
```

#### 删除 Markdown
```
- 点击"删除"按钮
- 确认对话框
- 调用 Appwrite Database.deleteDocument()
- 从列表中移除
```

## 🔐 安全性考虑

### 6.1 数据安全
- ✅ 用户密码使用 Appwrite 加密存储
- ✅ Session 存储在 chrome.storage.local
- ✅ 每个 Markdown 文档关联用户 ID，确保数据隔离
- ✅ 使用 HTTPS 通信
- ✅ XSS 防护（已有）

### 6.2 Appwrite 安全配置
- ✅ 启用用户认证
- ✅ 设置权限规则（Collection Permissions）
  - `markdowns` Collection:
    - Read: 用户只能读自己的文档
    - Write: 用户只能写自己的文档
- ✅ 启用访问日志

## 📦 集成方式

### 7.1 Appwrite SDK 集成

#### 方案 A: 通过 CDN 引入（推荐）
```html
<script src="https://unpkg.com/appwrite@14.0.0"></script>
```

#### 方案 B: 打包到扩展
- 下载 Appwrite SDK
- 放入 `lib/` 目录
- 在 manifest.json 中引用

### 7.2 配置管理
```
方式1: 硬编码（不推荐）
- 直接在代码中写入配置
- 优点：简单
- 缺点：不安全、不灵活

方式2: 环境变量
- 在构建时注入配置
- 优点：安全
- 缺点：需要构建流程

方式3: 用户配置（推荐）
- 首次使用时提示用户输入配置
- 优点：灵活、安全
- 缺点：需要额外配置步骤
```
